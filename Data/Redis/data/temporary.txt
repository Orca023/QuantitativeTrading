微軟視窗作業系統 ( Operating System: Acer-NEO-2023 Window11 x86_64 Inter(R)-Core(TM)-m3-6Y30 )

啓動 Redis 資料庫的服務端「redis-server」應用時，需讀入自定義配置文檔，設定自定義的啓動參數：
C:\> C:\QuantitativeTrading\Redis\Redis-3.2.100\redis-server.exe C:/QuantitativeTrading/Data/Redis/configuring_redis_with_option_files.conf

注意：

啓動 Redis 資料庫伺服器之前，需在自定義指定的配置文檔「configuring_redis_with_option_files.conf」裏，需要修改條目參數爲如下值：
dir C:/QuantitativeTrading/Data/Redis/data/
logfile "C:/QuantitativeTrading/Data/Redis/server_log.txt"

對於 Android-Termux-Ubuntu 系統，假設持久化保存的自定義路徑預設爲「/home/QuantitativeTrading/Data/Redis/data/」，則需在啓動 Redis 資料庫伺服器之前，在自定義指定的配置文檔「configuring_redis_with_option_files.conf」裏，需要修改條目參數爲如下值：
dir /home/QuantitativeTrading/Data/Redis/data/
logfile /home/QuantitativeTrading/Data/Redis/redis-server.log
pidfile /home/QuantitativeTrading/Data/Redis/redis-server.pid

修改工程空間路徑爲 Redis 應用可讀可寫可執行：
root@localhost:~# sudo chown redis:redis /home/QuantitativeTrading/Data/Redis -R
修改工程空間路徑權限爲所有用戶可讀可寫可執行：
root@localhost:~# sudo chmod 755 /home/QuantitativeTrading/Data/Redis
修改文檔權限爲所有用戶可讀可寫：
root@localhost:~# sudo chmod 640 /home/QuantitativeTrading/Data/Redis/configuring_redis_with_option_files.conf
root@localhost:~# sudo chmod 640 /home/QuantitativeTrading/Data/Redis/redis-server.log
root@localhost:~# sudo chmod 640 /home/QuantitativeTrading/Data/Redis/redis-server.pid
